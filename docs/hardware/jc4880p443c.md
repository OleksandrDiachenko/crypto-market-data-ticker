# Board: JC4880P443C_I_W

GUITION/JCZN 4.3" ESP32-P4 display board. Core module `JC-ESP32P4-M3`
(ESP32-P4 + ESP32-C6 companion SoC for Wi-Fi/Bluetooth).

## Main controller and memory (Imported)

- SoC: ESP32-P4, dual-core RISC-V.
- 768 KB HP L2MEM, 32 KB LP SRAM, 128 KB HP ROM.
- PSRAM: 32 MB. Flash: 16 MB.

## Wireless (Imported)

- ESP32-P4 has no native Wi-Fi/Bluetooth; connectivity goes through the
  ESP32-C6 companion SoC (ESP-Hosted or a board-specific bridge).
- Do not assume direct `esp_wifi_*` calls work on the P4 without the C6 link
  configured.

## Display (Imported)

- Panel: 4.3" TFT IPS, 480x800, RGB565.
- Driver IC: ST7701S over MIPI DSI, 2 lanes, 500 Mbps lane bitrate.
- DPI timing: clock 34 MHz; horizontal active 480 / back porch 42 / pulse
  width 12 / front porch 42; vertical active 800 / back porch 8 / pulse
  width 2 / front porch 166. Frame buffers: 1.
- Backlight GPIO: `GPIO_NUM_23`. LCD reset GPIO: `GPIO_NUM_5`.
- **The default init command sequence in the registry `esp_lcd_st7701`
  component does not work on this board.** A board-specific ST7701 init
  sequence is required (passed via `st7701_vendor_config_t.init_cmds`).
- Do not use the registry `ST7701_480_360_PANEL_60HZ_DPI_CONFIG()` macro —
  it does not match this board's timing. Build an explicit
  `esp_lcd_dpi_panel_config_t` instead.

## Touch (Imported)

- Controller: GT911, I2C.
- I2C pins: SDA `GPIO_NUM_7`, SCL `GPIO_NUM_8`.
- Reset/interrupt: `GPIO_NUM_NC` in the known-working 480x800 configuration
  (i.e. not wired to a dedicated GPIO in that setup).

## Audio (Imported, out of scope until a later phase)

- Codec: ES8311 on the shared I2C bus, I2S audio.
- I2S pins: BCLK `GPIO_NUM_12`, MCLK `GPIO_NUM_13`, LRCLK `GPIO_NUM_10`,
  DOUT `GPIO_NUM_9`, DIN `GPIO_NUM_48`. PA enable: `GPIO_NUM_11`.

## microSD / SDMMC (Imported, out of scope until a later phase)

- D0-D3: `GPIO_NUM_39/40/41/42`. CLK: `GPIO_NUM_43`. CMD: `GPIO_NUM_44`.

## Known pitfalls (Imported)

- PSRAM must run at 200 MHz (`CONFIG_SPIRAM_SPEED_200M=y`). At 20 MHz this
  produced `lcd.dsi.dpi: can't fetch data from external memory fast enough,
  underrun happens`. This project's `sdkconfig.defaults` already sets
  200 MHz.
- This MIPI DPI panel does not support hardware rotation:
  `esp_lcd_panel_swap_xy(...): swap_xy is not supported by this panel`.
  Keep `.swap_xy = false` and `.sw_rotate = true` in the LVGL port config.
- DMA2D has triggered an ISR assert on this board on ESP-IDF 5.5.x with the
  registry ST7701 path. Keep `.flags.use_dma2d = false` until revalidated.
- A `ledc: GPIO 23 is not usable, maybe conflict with others` warning during
  backlight PWM setup is a separate, lower-priority issue from DSI underrun —
  if the panel is otherwise lit, treat brightness/PWM as a follow-up.
- Two USB-C ports exist (full-speed and high-speed); flashing/serial
  behavior can differ between them. If flashing fails, try the other port
  and note in the validation report which port was used for what.

## Recommended component dependencies (Imported)

```yaml
dependencies:
  espressif/esp_lcd_st7701:
    version: "^2.0.2"
    public: true
  espressif/esp_lcd_touch_gt911:
    version: "^1"
    public: true
  espressif/esp_lvgl_port:
    version: "^2"
    public: true
  espressif/esp_codec_dev:
    version: "1.2.*"
    public: true
```

LVGL: prefer v9. Use a partial DMA draw buffer (`buff_dma = true`,
`buff_spiram = false`) during bring-up rather than a full-screen or
PSRAM-backed buffer.
